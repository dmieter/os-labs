Лабораторная работа №5 выполняется индивидуально или бригадами по 2 человека. При выполнении бригадой необходимо дополнительно подготовить документацию на разработанное ПО (с примерами работы, скриншотами и описанием основных идей).
 
1. Базовое решение для успешной сдачи лабораторной работы.
Разработать Операционную систему на языке ассемблер под архитектуру х86 (достаточно одного загрузочного сектора), которая при загрузке осуществляет вывод собственного названия и лого в ASCII графике, например:
<pre>
 __   __        __   ___          __   __  
|__) /  \ |    / _` |__  |\ |    /  \ /__` 
|__) \__/ |___ \__> |___ | \|    \__/ .__/ 
</pre>

2. Для получения Хорошей оценки необходимо выполнить пункт 1 и добавить элементы взаимодействия с пользователем. Например, простой диалог, проведение вычислений на основе вводимых с клавиатуры данных: y/n, имя, число и т.п. 

3. Для получения отличной оценки разработанная Операционная система должна включать взаимодействие с пользователем (пункт 2), а также реализовать что-либо из перечисленного:  
  - игровая механика (ветвление диалога, карточные игры, двумерные игры, перемещение по лабиринтам, flappy bird и др.) в обоснованно упрощенном варианте;
  - особое цветовое и стилевое оформление интерфейса;
  - загрузка дополнительных (и необходимых для работы ОС) данных с носителя (например, из второго сектора);
  - что-то еще?
					   
*Для защиты необходимо:*
- продемонстрировать работоспособность преподавателю, внести небольшие изменения в код по запросу преподавателя
- предоставить в электронном виде документацию/отчет, если в бригаде более одного человека
